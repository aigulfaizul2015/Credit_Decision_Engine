# Atlas Underwriting — Automated Credit Analysis System

Система кредитного анализа юридических лиц на базе n8n.  
Принимает заявку с ИНН компании и параметрами кредита, проводит полный анализ и формирует кредитное заключение.

---

## Архитектура

```
WF_Credit_Form          → точка входа, веб-форма
WF_Credit_Decision_Engine → основной пайплайн анализа
WF_Audit_Logger         → логирование каждого шага в Google Sheets
WF_Status_API           → API для проверки статуса заявки
```

---

## WF_Credit_Form

Отдаёт клиенту HTML-страницу с формой заявки.

При открытии `/webhook/credit-form` генерирует и возвращает готовый HTML: форму ввода (ИНН, КПП, сумма и срок кредита), блок отслеживания статуса с автообновлением каждые 10 секунд, и рендер финального кредитного заключения.

---

## WF_Audit_Logger

Вспомогательный workflow — логирует каждый шаг обработки заявки.

Вызывается из `WF_Credit_Decision_Engine` после каждого значимого шага. Получает `applicationId`, название шага (`tool`) и его результат (`output`), проставляет временну́ю метку и записывает строку в Google Sheets. Эти данные используются `WF_Status_API` для отслеживания прогресса.

---

## WF_Status_API

REST API для опроса статуса заявки.

`POST /webhook/application-status` с телом `{"applicationId": "..."}` — читает все строки из Google Sheets по данному ID, находит последнюю по дате, определяет тип завершения (`SUCCESS`, `DADATA_NOT_FOUND`, `DADATA_API_ERROR`, `VALIDATION_ERROR`) и возвращает JSON с текущим шагом, флагом завершённости и, если готово, финальным отчётом.

---

## WF_Credit_Decision_Engine

Основной пайплайн. Запускается через `POST /webhook/credit-application`.


---

### Шаги пайплайна

| Tool ID | Название | Что происходит |
|---|---|---|
| `01_Input_Form` | Приём заявки | Парсит входящие данные формы: ИНН, КПП, сумму и срок кредита, фиксирует дату запроса. |
| `02_Input_Validation` | Валидация данных | Проверяет корректность полей: ИНН — 10 цифр, КПП — 9 цифр, сумма > 0, срок 1–120 месяцев, только цифры. При ошибке формирует список ошибок. |
| `03_Input_Validation_Error` | Ошибка валидации | Финальный шаг при провале валидации — логируется сообщение об ошибке, обработка останавливается. |
| `03_Company_Profile_Builder` | Профиль компании (Dadata) | По ИНН или ИНН и КПП запрашивает данные из Dadata API: название, ОГРн, ОКВЭД, адрес, ФИО и должность руководителя, статус компании, дату регистрации. |
| `03_DADATA_NOT_FOUND` | Компания не найдена | Dadata вернула пустой список — компания не существует или не найдена. Обработка останавливается. |
| `03_DADATA_API_ERROR` | Ошибка Dadata API | Dadata вернула ошибку — проблема на стороне внешнего сервиса. Обработка останавливается. |
| `04_Company_Profile_Data_Builder` | Обогащение профиля | Обрабатывает профиль компании из ответа Dadata: возраст компании, категория возраста, нормализация адреса. |
| `05_Company_Profile_Normalization_Builder` | Нормализация данных | Приводит все поля профиля к строгим типам, убирает лишние поля, готовит данные для аналитических модулей. |
| `06_Financial_Agent_AI_Agent` | Финансовый AI-агент | AI-агент с веб-поиском ищет актуальные финансовые данные компании: выручку, прибыль, задолженности — из открытых источников. |
| `06_Registration_Assessment` | Проверка регистрации | Анализирует регистрационные риски: статус компании (ликвидация, реорганизация), адрес, директор, ОКВЭД, дата регистрации. |
| `06_Compliance_Assessment` | Комплаенс-оценка | Проверка рисков по статусу, массовому адресу или директору, полноты информации, юридические риски (ЕГРЮЛ, банкротсво). |
| `06_FRAUD_ALLEGATION_FOUND` | Репутация: мошенничество | AI-поиск публичных обвинений в мошенничестве. |
| `06_REPUTATIONAL_SCANDAL_FOUND` | Репутация: скандал | AI-поиск публикаций о репутационных скандалах. |
| `06_NEGATIVE_MEDIA_COVERAGE` | Репутация: негативные СМИ | AI-поиск негативных упоминаний в медиа без конкретных обвинений. |
| `06_PUBLIC_LITIGATION_MENTIONED` | Репутация: судебные споры | AI-поиск публичной информации о судебных разбирательствах. |
| `06_PUBLIC_ACCUSATION_FOUND` | Репутация: публичные обвинения | AI-поиск обвинений без признака мошенничества. |
| `06_POSITIVE_MEDIA_COVERAGE` | Репутация: позитивные СМИ | AI-поиск позитивных упоминаний в медиа. |
| `06_INDUSTRY_AWARD_FOUND` | Репутация: отраслевые награды | AI-поиск наград или признаний в отрасли. |
| `06_PUBLIC_RECOGNITION_FOUND` | Репутация: общественное признание | AI-поиск иных форм положительной публичной репутации. |
| `07_Financial_Assessment` | Финансовая оценка | На основе данных AI-агентов оценивает финансовое состояние: достаточность выручки, наличие прибыли, долговую нагрузку, способность обслуживать кредит. |
| `07_Credit_Assessment` | Кредитная оценка | Формирует кредитные findings: превышение суммы кредита над выручкой (`LOAN_EXCEEDS_REVENUE`), отсутствие прибыли (`NO_PROFIT_SUPPORT`) и другие кредитные риски. |
| `08_Reputation_Evidence_Aggregation` | Агрегация репутационных данных | Собирает все репутационные findings из параллельных веток в единый объект с источниками, цитатами, ссылками и уровнем уверенности. |
| `08_Risk_Assessment` | Общая оценка рисков | Рассчитывает итоговый `risk_score` и уровень риска (`LOW`/`MEDIUM`/`HIGH`) на основе всех собранных оценок. |
| `09_Policy_Assessment` | Проверка кредитной политики | Применяет правила политики: автоматическое одобрение, отклонение (ликвидация компании, признак мошенничества) или направление на ручное рассмотрение (risk_score ≥ 50). |
| `10_Decision_Explanation` | Объяснение решения | Формирование человекочитаемое объяснения решения: сильные стороны заявки, факторы риска. |
| `11_Credit_Committee_Chairman` | Кредитный комитет | Финализирует решение с позиции председателя кредитного комитета — выносит итоговый вердикт (`APPROVE`/`REVIEW`/`DECLINE`) с учётом всех материалов. |
| `12_Final_Credit_Report` | Финальный кредитный отчёт | Собирает все оценки в единый структурированный объект кредитного заключения. |
| `13_Client_Report_Builder` | Клиентский отчёт | Формирует финальный JSON-отчёт для отображения клиенту: решение, риск-скор, сильные стороны и факторы внимания, подтверждающие материалы с источниками. |
